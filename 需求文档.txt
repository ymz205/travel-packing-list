一、项目基本信息
产品名称：旅行行李自动规划工具（纯本地离线版）
运行模式：纯前端离线网页，无网络请求、无后端接口、无 API 调用、无密钥配置
核心定位
打开网页即可使用，依靠前端内置数据库 + 规则匹配算法。用户填写出行信息后，自动生成行李打包清单；同时根据所选城市，自动展示目的地美食、风土人情、打卡景点，全程本地运行，断网也能正常使用。
二、技术实现要求（硬性约束）
交付产物：单个独立 HTML 文件，整合 HTML+CSS+JavaScript，无外部依赖，双击文件就能在浏览器打开运行。
禁止使用：fetch、axios、网络请求、第三方接口、大模型 API。
所有基础数据（物品清单、城市美食、景点、风土资料）全部硬编码写在 JS 数组中。
数据持久化：使用浏览器 LocalStorage 实现本地缓存，刷新页面数据不丢失。
页面布局：响应式自适应，同时兼容 PC 电脑端与手机移动端。
UI 风格：简约旅行主题，按钮带有点击交互效果，支持浅色 / 深色主题一键切换。
三、页面整体布局结构
顶部：标题 + 主题切换按钮
中部：出行信息填写表单区域
主体内容：双 Tab 标签栏切换
Tab1：行李打包清单模块（默认打开）
Tab2：目的地游玩指南模块
底部：操作按钮（生成方案、一键重置）
四、详细功能需求
模块 1：出行信息表单（交互录入页面）
输入项：
目的地：下拉选择热门城市，同时支持手动输入自定义城市
出行天数：数字输入框
出行人数：数字输入框
出行季节：下拉选项（春、夏、秋、冬）
出行场景：下拉选项（休闲旅游、商务出差、海边度假、山野露营、回乡探亲）
按钮：【生成方案】【一键重置】
模块 2：Tab1 行李自动规划（原有基础功能全部保留）
本地内置物品资源库，分类存储：衣物、洗护用品、证件资料、电子产品、常备药品、杂物。
自动匹配逻辑：
根据出行天数、季节、出行场景，自动筛选物品，并自动计算携带数量。
清单交互能力：
自动生成行李列表
支持物品勾选、单独删除条目
支持手动自定义新增物品
支持一键复制清单文本
数据存储：行李清单自动存入浏览器缓存，刷新页面内容不消失。
模块 3：Tab2 目的地游玩指南（本次新增核心功能）
当选定城市并点击生成方案后，自动读取本地城市数据库，展示三大板块内容：
特色美食推荐
展示城市正餐美食 + 特色小吃，附带简短文字介绍，支持手动编辑、新增美食条目。
风土人情介绍
包含当地气候特点、民俗文化、出行风俗习惯、旅行注意事项。
网红打卡景点清单
景点分为三类：自然风光、人文古迹、商业街区。
联动附加功能：
结合目的地 + 所选季节，自动生成穿衣建议，同步更新衣物类行李推荐；自动匹配出行贴士（防晒、防雨、海边装备、户外露营装备提醒）。
编辑权限：用户可以自行修改、添加景点与美食内容，修改内容自动本地缓存。
模块 4：通用交互优化功能
Tab 标签无缝来回切换，两个模块数据互不丢失。
浅色 / 深色模式一键切换，全局样式同步变更。
一键重置功能：清空表单、清空行李清单、清空游玩信息、清空本地缓存。
整页自适应，手机端自动排版，不会出现横向滚动条。
全程无任何网络请求，所有数据仅保存在本地浏览器。
五、完整业务流程
用户打开 HTML 网页，自动加载上次缓存数据（无缓存则为空）。
在表单填写目的地、天数、人数、季节、出行场景。
点击【生成方案】：
Tab1 自动运行匹配算法，生成行李打包清单；
Tab2 自动加载该城市美食、风土人情、打卡景点，同时生成穿衣建议。
用户手动修改行李条目、编辑游玩信息，所有改动自动缓存。
可复制行李清单文本保存到本地。
点击【一键重置】，清空全部表单与数据。
六、给到代码 Agent 的执行指令（直接复制）
plaintext
开发独立单文件HTML网页，内嵌css与js，不需要引入外部文件，双击即可离线打开运行。
严格遵守以下约束：
1. 禁止所有网络请求，不调用任何API接口，所有城市数据、物品清单全部写死在JS数组内；
2. 使用LocalStorage做本地数据缓存，刷新页面数据保留；
3. 页面分为表单区+双Tab栏：Tab1行李清单，Tab2目的地游玩指南；
4. 表单字段：目的地（下拉+自定义输入）、出行天数、人数、季节、出行场景；
5. 编写规则算法，根据出行条件自动筛选物品并计算数量，行李清单支持勾选、删除、自定义新增、一键复制；
6. 选中城市后，在Tab2展示：特色美食、风土人情、分类打卡景点，并且支持手动编辑内容；
7. 根据城市+季节自动生成穿衣建议与出行装备贴士，联动优化行李衣物推荐；
8. 增加浅色/深色主题切换按钮、一键重置按钮；
9. 页面做响应式布局，PC端与移动端正常显示，旅行简约UI，按钮增加点击反馈；
10. 最终只输出一整段完整HTML代码，无拆分文件，保证离线零网络正常运行。
七、归档精简版 PRD（文档存档用）
1. 产品目标
制作离线前端网页应用，依靠本地内置数据与匹配算法，实现智能行李清单生成，并配套目的地美食、风俗、景点查询，全程离线运行，数据本地缓存。
2. 页面模块
出行信息表单页面
Tab1：行李自动打包清单模块
Tab2：目的地旅游信息模块
3. 功能要点
自动生成行李清单，支持手动编辑与文本复制；
选定城市自动加载美食、风土人情、分类打卡景点，内容可自定义修改；
季节 + 城市联动生成穿衣方案与出行贴士；
Tab 切换、深浅主题、移动端自适应、本地缓存保存数据；
一键重置全部内容。
4. 技术规范
纯前端单 HTML 文件，无网络请求，所有数据写在 JS 数组，LocalStorage 持久化，禁止后端与接口调用。
